# Editorial Ethics

Read this when walking a user through Area 4 (Editorial Standards) during setup, or when updating standards on an existing preset.

This file covers foundational frameworks the user might want to draw on, and rules for converting verbose ethics language into short, directive entries that work inside a preset.

---

## Frameworks you can offer

Use these only as a gap-filler **after** the user has shared their newsroom's own standards. Their newsroom rules are primary.

### IFJ Global Charter of Ethics for Journalists (2019)

The most widely referenced international framework. 16 articles plus a preamble, adopted at the 30th IFJ World Congress in Tunis. Covers respect for facts, protection of sources, editorial independence, non-discrimination, and professional solidarity.

Link: https://www.ifj.org/who/rules-and-policy/global-charter-of-ethics-for-journalists

Use when the journalist wants a comprehensive international baseline, or when they're a freelancer without a newsroom framework to start from.

### SPJ Code of Ethics (US-oriented)

The Society of Professional Journalists' code. Four principles: Seek truth and report it; Minimise harm; Act independently; Be accountable and transparent. Less prescriptive than the IFJ Charter but widely cited in US newsrooms.

Link: https://www.spj.org/ethicscode.asp

Use when the journalist is US-based and wants a locally familiar framework.

### Ofcom Broadcasting Code (UK, broadcast-specific)

UK regulatory code for broadcasters. Covers accuracy, impartiality, fairness, privacy, and harm. Legally binding on UK broadcasters.

Link: https://www.ofcom.org.uk/tv-radio-and-on-demand/broadcast-codes/broadcast-code

Use when the journalist works for a UK broadcaster or wants broadcast-aware standards.

### Other regional/national frameworks

If the journalist is in a country with a strong local framework (e.g., the NUJ Code of Conduct in the UK, the APME Statement of Ethical Principles in the US, national press council codes in various countries), and they reference it, use that instead of a global one. Don't substitute a framework the journalist didn't name.

## Conversion rules

Newsroom ethics policies are usually written as prose. A preset needs short, directive, unambiguous rules. Convert as follows.

### Rule 1: Every rule becomes an imperative

- **Source:** "Journalists should strive to verify information from multiple sources before publication."
- **Converted:** "Verify every fact against at least two independent sources before use."

### Rule 2: Replace soft verbs with specific actions

- "Strive to", "endeavour to", "aim to" → direct action verbs
- "Where possible", "generally" → specify the exception condition, or drop the hedge

### Rule 3: Attach a verification step where possible

- **Source:** "Quotes should accurately reflect what the source said."
- **Converted:** "Check every verbatim quote against the original recording before use. AI hallucinates quote wording."

### Rule 4: Collapse related rules

- If three paragraphs all say "be careful with sources," compress to one rule with specific actions.

### Rule 5: Drop rules that don't apply to scripts

Newsroom policies cover a lot — hiring practices, complaints procedures, social media conduct outside work, advertising separation. For a script preset, keep only what applies to the act of drafting a script.

Keep:
- Accuracy and verification rules
- Source protection rules
- Attribution rules
- Impartiality / balance rules (if the newsroom has them)
- Correction and transparency rules
- Rules about specific sensitive topics on the journalist's beat

Drop:
- HR, hiring, or workplace rules
- Advertising and commercial separation (unless scripts could breach it)
- Internal complaints procedures
- Social media conduct outside scripting

### Rule 6: Target ~8–15 rules

More than 15 and the preset becomes unwieldy. Fewer than 8 and you've probably cut too much. If a newsroom's policy genuinely has 25 core rules that all apply to scripting, ask the user to prioritise the top 15.

### Rule 7: Flag anything that contradicts other parts of the preset

If a standard contradicts the voice profile, style guide, or publication context, flag it during the mini-review. Example: a standard says "never use first person in copy" but the voice profile uses "I". This is a contradiction the user needs to resolve deliberately.

## What to warn the user after conversion

> ⚠️ I've converted your newsroom standards into short directive rules. Review them carefully — I may have:
> - Misinterpreted nuanced policy language
> - Dropped rules I thought were inapplicable but that you want kept
> - Combined rules that you prefer separate
> - Lost context-specific exceptions
>
> Your standards are editorially load-bearing. Don't rubber-stamp this — read each rule and confirm it represents your newsroom's policy accurately.

## Special case: freelancers with no newsroom

If the user is a freelancer with no parent newsroom standards:

1. Offer the IFJ Global Charter as a baseline (or a regional framework if they're in a country with one).
2. Extract the ~10 rules that apply most directly to scripting (truth, verification, sources, attribution, corrections, non-discrimination, independence).
3. Add any personal standards the freelancer has developed themselves.
4. Note in the preset that these are personal standards derived from [framework name], not tied to a specific newsroom.

---

## Special case: journalist says "I don't have standards"

Some journalists haven't thought explicitly about this before. If that happens:

1. Don't move on without something. Standards affect every script.
2. Walk them through a quick minimum: "Let's cover the basics — how do you handle facts you can't verify? What about quotes? What do you do if you find an error after publishing?"
3. Offer the IFJ Charter as a starting point they can adopt as-is or modify.
4. Make clear that the standards can be updated later as they develop a clearer view of their own ethical line.
