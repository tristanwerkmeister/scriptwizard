# Contributing to Script Wizard

Thanks for your interest in contributing! Script Wizard is a prompt-engineering framework for journalists, so contributions tend to be about improving prompts, workflows, and editorial guidance rather than traditional code.

## How to contribute

1. **Open an issue first.** Before submitting a pull request, open an issue describing what you'd like to change and why. This helps avoid duplicate work and lets us discuss the approach.

2. **Submit a pull request.** Once the approach is agreed, fork the repo, make your changes, and open a PR. Keep changes focused — one issue per PR where possible.

## What's welcome

- Improvements to prompt instructions (clearer wording, better edge-case handling)
- New or improved reference materials (voice profile guidance, review checklists, etc.)
- Bug reports (prompts that produce unexpected or incorrect behaviour)
- Documentation improvements
- Support for additional AI tools or platforms

## Editorial and ethical content

Changes to editorial ethics frameworks (`references/editorial-ethics.md`), editorial standards guidance, or sensitivity warnings should **always be discussed in an issue first**. These sections are editorially load-bearing — a poorly worded change could affect how journalists handle sensitive content. Please explain the reasoning behind proposed changes.

## Style

- Keep prompt language directive, not descriptive ("Do X" rather than "The system should X")
- Match the tone and formatting of existing files
- Test your changes by running the prompts in an AI assistant if possible
